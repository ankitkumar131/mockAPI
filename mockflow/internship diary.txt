General Guidelines for Industry Project and Internship: 
Parvatibai Genba Moze Collage of Engineering, Wagholi, Pune has introduced one semester 2 
months internship/ field project with an aim to provide exposure to industrial environment 
considered necessary to groom students for their profession. Following are some general 
guidelines, Do’s, and Don’ts to be followed. 
The objectives of the Industrial Training/ Internship are:  
1. To provide an opportunity for students to familiarize with the industry of their 
discipline, experience, work, culture and discover the organizations within the industry. 
Students will acquire interpersonal skills through meeting with professionals in their 
field of study. 
2. To provide an opportunity for students to acquire practical skills and experience 
working on projects alongside industry experts. 
3. To provide an opportunity for students to discover grass root problems and fundamental 
issues in industry with a view to take up major project and development of innovative 
solutions. 
Expected Learning Outcomes after the Training/Internship are: 
• Learn team work and professionalism 
• Apply SDLC to Project 
• Apply communication and presentation skills 
• Recognize the importance of documentation 
• Ability to work under supervision and directions. 
• Improved Communication & Interpersonal Skills. 
• Work Ethics of the company/industry. 
• Efficiently completing tasks, fostering good relationship with seniors and subordinates. 
• Exposure to latest technology applications to the specific discipline. 
• Identification of relevant problems in the industry and innovative solutions. 
• Assured opportunities in Top Companies 
Stages in Field Project/ Internship 
Stage 01: Orientation in the company (15 Days). 
• During the first months of internship, the intern shall undergo an extensive orientation 
programme in industry. 
• The intern should carry out a detailed study of products /services, processes offered by 
the industry. 
• Intern should identify the suppliers, customers and competitors of the industry. 
• Intern should understand the organizational structure, vision, mission policies and top 
management of the industry.

control etc.

Introduction to Automation Testing 
During the fourth week of my internship, I was introduced to automation testing 
for the ThermoFisher project. While manual testing helps verify functionality 
through human interaction, automation testing allows repetitive test scenarios to 
be executed automatically using scripts. This helps improve testing efficiency and 
ensures that application features work consistently across multiple executions. 
Learning Automation Framework 
The project initially used Protractor for automation testing of the web 
application. Protractor is commonly used for testing applications developed using 
Angular because it provides built-in support for Angular components and 
asynchronous operations. 
During this week, I learned how the automation framework was structured and 
how automated test scripts were organized within the project. 
Understanding Automation Test Script Structure 
Automation test scripts followed a structured format which made the tests 
organized and easy to maintain. The basic structure of the test scripts included: 
• Locators used to identify UI elements on the web page 
• Creation of test suites to group related test cases 
• Writing test scenarios using describe blocks 
• Implementing individual test cases inside it blocks 
In the automation scripts, XPath locators were commonly used to identify 
specific elements on the webpage such as buttons, input fields, and links. 
Writing and Understanding Test Cases 
During this week, I reviewed existing automation test scripts to understand how 
automated tests were written and executed. These scripts simulated user 
interactions such as clicking buttons, entering data into fields, and navigating 
between pages.

Migration of Automation Testing Scripts 
During the eighth week of my internship, I was assigned a task related to the 
migration of automation testing scripts used in the ThermoFisher project. The 
existing automation framework was initially built using Protractor, but the team 
decided to migrate the automation scripts to Playwright to improve performance, 
reliability, and long-term maintainability. 
Understanding the Need for Migration 
Protractor was previously widely used for testing Angular applications, but with 
the advancement of modern testing tools, the team decided to adopt Playwright 
because it provides faster execution, better browser support, and improved 
automation capabilities. 
As part of this migration process, existing automation scripts written in Protractor 
needed to be converted into Playwright-compatible scripts. 
Migration Process 
For every batch, the process involved: 
The migration was carried out in multiple batches to ensure that the process was 
organized and manageable. Each batch contained a group of automation test 
scripts that needed to be updated. 
• Reviewing the existing Protractor automation scripts 
• Understanding the test logic and scenarios implemented in those scripts 
• Rewriting the automation code in Playwright format 
• Updating locators and test structure where required 
Updating Test Structure 
During the migration process, the structure of the automation tests was adjusted 
to match the Playwright framework. The new scripts maintained the same testing 
logic but used Playwright’s syntax and functions for executing browser actions 
and verifying results.
